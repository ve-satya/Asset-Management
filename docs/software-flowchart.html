Compliance Engine

How the system decides if you are Under Licensed, Over Licensed, Compliant, or N/A

💡 Compliance is computed live — not stored. Every time you load the software list the system counts active installations vs total allowed licenses and applies this logic.
flowchart TD TRIGGER(["GET /api/softwares\n→ for each software row"]):::oval TRIGGER --> STEP1["Count active installations\ninstallCount = active\ninstallation records\nfor this software"]:::box STEP1 --> STEP2["Sum license allowances\ntotalAllowed = SUM of\ninstallationsAllowed\nacross all active licenses"]:::box STEP2 --> CHECK1{"Software Type:\nenableCompliance\n= true?"}:::decision CHECK1 -->|"❌ No\n(Freeware, Excluded,\nUnidentified...)"| NA["⚪ N/A\nCompliance tracking\ndisabled for this type\nNo action needed"]:::na CHECK1 -->|"✅ Yes\n(Managed, Prohibited...)"| CHECK2{"installCount\nvs\ntotalAllowed"}:::decision CHECK2 -->|"Installed > Allowed\nEg: 8 installs, 5 allowed"| UNDER["🔴 Under Licensed\nYou have more installations\nthan you have licenses for.\nRisk: Non-compliant!"]:::under CHECK2 -->|"Allowed > Installed\nEg: 5 installs, 8 allowed"| OVER["🟠 Over Licensed\nYou bought more licenses\nthan you are using.\nWaste: reduce next renewal"]:::over CHECK2 -->|"Installed = Allowed\nEg: 5 installs, 5 allowed"| COMP["🟢 Compliant\nPerfect match.\nAll good!"]:::comp UNDER & OVER & COMP & NA --> BADGE["Shown as colored badge\nin the software table row\n\nAlso feeds the\n📊 Software Summary\nDashboard pie chart"]:::output classDef oval fill:#7c3aed,color:#fff,stroke:#5b21b6 classDef box fill:#f5f3ff,stroke:#7c3aed,stroke-width:1.5px,color:#3b0764 classDef decision fill:#fdf4ff,stroke:#9333ea,stroke-width:2px,color:#581c87 classDef na fill:#f1f5f9,stroke:#94a3b8,stroke-width:2px,color:#475569 classDef under fill:#fef2f2,stroke:#dc2626,stroke-width:2px,color:#991b1b,font-weight:bold classDef over fill:#fff7ed,stroke:#ea580c,stroke-width:2px,color:#7c2d12,font-weight:bold classDef comp fill:#f0fdf4,stroke:#16a34a,stroke-width:2px,color:#14532d,font-weight:bold classDef output fill:#eff6ff,stroke:#3b82f6,stroke-width:2px,color:#1e3a8a

Software Licenses — Create, Allocate & Track

How license counts work and how Available is calculated

💡 Available = installationsAllowed − allocated. This number tells you how many more installations you can assign before running out of licenses.
flowchart LR subgraph CREATE["➕ Creating a License"] direction TB CF["License Form fields:\n• Software (required)\n• License Type\n• License Option\n• Purchased count\n• Installations Allowed\n• License Key\n• Cost, Vendor\n• Agreement (optional)\n• Expiry Date"] CF --> CAPI["POST /api/global-software-licenses\n→ 201 Created"] CAPI --> CALC["System stores:\npurchased = X\ninstallationsAllowed = Y\nallocated = 0\navailable = Y - 0 = Y"] end subgraph ASSIGN["🔗 Assigning a License\nto an Installation"] direction TB IA["Software Installation\nrecord created:\nPUT /api/software-installations/:id\n{ licenseId: X }"] IA --> RECALC["allocated count increases\navailable = allowed - allocated\nNew available = Y - 1"] RECALC --> LIMIT{"available\n= 0 ?"} LIMIT -->|"No seats left"| WARN["⚠️ Warning shown:\n'No available licenses'\nCannot assign more"] LIMIT -->|"Still seats"| OK["✅ Assignment recorded\nInstallation linked to license"] end subgraph TRACK["📊 Tracking Columns in Table"] direction TB TC["Columns visible per row:\n\nPurchased — how many you bought\nInstalled — actual install count\nAllocated — assigned to licenses\nAvailable — remaining to assign\nExpiry — days until expiry\n\nExpiry label:\n• 'X days ago' → expired\n• 'Today' → expiring now\n• 'X Days' → future"] end subgraph SOFTDELETE["🗑️ Soft Delete a License"] direction TB SD["DELETE /api/global-software-licenses/:id\nSets isActive = false\nLicense hidden from lists\nHistory preserved in DB"] end CREATE --> ASSIGN ASSIGN --> TRACK TRACK --> SOFTDELETE

Software Summary Dashboard — Data Flow

How the dashboard reads data and what each chart shows

💡 The dashboard makes one API call that does all the number-crunching on the server. Changing the Manufacturer filter re-runs the call and all charts update together.

All softwares + types + license counts\n2. All active licenses\n3.
